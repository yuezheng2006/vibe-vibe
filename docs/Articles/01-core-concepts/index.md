# 核心概念与范式演进

如果你对这场变化的理解还停留在“AI 能帮我写代码”，那后面很多讨论都会显得像零散技巧，而不是同一张地图上的不同坐标。这个栏目要先解决的，正是地图问题：什么是 AI Agent，什么是 Agentic Engineering，Vibe Coding 真正改变的又是什么。

更重要的是，这些文章讨论的不是“写代码这件事是否会变快”，而是开发者角色正在怎样迁移——从亲手实现每个细节，转向定义目标、组织上下文、拆解任务、审查结果，并为系统质量负责。换句话说，变化的核心不是自然语言替代键盘，而是软件生产方式开始从“手工制作”转向“编排系统”。

这一栏也有一个必须尽早建立的判断：快，不等于随便。AI 把实现门槛压低了，但它同样放大了规格不清、验证不足和质量纪律松动的代价。读完这里，你应该获得三件东西：对核心概念的清晰定义、对开发者新角色的基本认识，以及对“速度必须受工程纪律约束”这件事的直觉。
