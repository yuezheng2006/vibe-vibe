# 关于 Vibe Coding 你需要知道的一切

<div class="article-meta">
📅 2025-02-01 | 👤 Zeb Hermann, Keith Messick | 🔗 <a href="https://vercel.com/blog/what-you-need-to-know-about-vibe-coding" target="_blank">查看原文</a>
</div>

Vibe coding 正在彻底改变我们的工作方式。以下是 Vercel 发布的《Vibe Coding 现状报告》中的核心要点。

2025 年 2 月，Andrej Karpathy [提出](https://x.com/karpathy/status/1886192184808149383?lang=en)了 _vibe coding_ 这个术语：一种全新的 AI 编程方式，"你完全顺应直觉，拥抱指数级增长，甚至忘记代码的存在。"

短短几个月后，vibe coding 已经彻底重塑了开发者和非开发者的工作方式。[超过 90%](https://github.blog/news-insights/research/survey-reveals-ais-impact-on-the-developer-experience/) 的美国开发者在使用 AI 编程工具，其他角色的采用率也在加速增长，英语已成为世界上增长最快的编程语言。

我们在最新的 [Vibe Coding 现状报告](https://v0.app/vibecoding)中详细探讨了这一转变。以下是几个关键要点。

## 从 No-code 到 Vibe Code

十五年前，no-code 工具让人们通过拖拽组件来构建应用。这标志着"所见即所得"（WYSIWYG）软件工具的开端，用户可以在视觉界面中编辑内容，直接看到最终效果。

现在，像 v0、Lovable 或 [Replit](https://v0.app/docs/compare/v0-vs-replit) 这样的 vibe coding 工具让这个过程变得更简单，进入了"所见即所得 2.0"时代：你*说*什么，就*得到*什么。任何人都可以即时生成应用、网站或工作流。无需经验。这正在改变各种规模组织中个人和团队的工作方式，改变项目的人员配置方式和个人的贡献方式。

![从 No-code 到 Vibe Code 的演进](/images/articles/2025-02/vibe-coding-no-code-to-vibe-code.png)

## Vibe Coding 的核心优势

大型公司现在更公开地采用 vibe coding。亚马逊推出了自己的 vibe coding 工具 [Kiro](https://kiro.dev/blog/introducing-kiro/)，[超过 30%](https://abc.xyz/assets/66/ae/c94682fc4137b5fb90a5d709ac4b/2025-q1-earnings-transcript.pdf) 的 Google 新代码由 AI 生成。但影响不仅限于科技巨头。

任何规模的团队现在都可以在几天而不是几个月内完成项目。事实上，2025 年 3 月，[Garry Tan 表示](https://www.cnbc.com/2025/03/15/y-combinator-startups-are-fastest-growing-in-fund-history-because-of-ai.html) 10 人的 AI 驱动团队现在可以完成曾经需要 100 名工程师的工作。Vibe coding 还将生产力单位从团队转移到个人。只需一个人就可以创建原型或构建产品。

更快的迭代、成本节约，以及想法与执行之间更少的鸿沟，都触手可及。唯一真正的门槛是能够清晰地描述你的想法。

## Vibe Coding 面向所有人

Vibe coding 重新定义了谁能编程、他们能创造什么，重塑了开发者的真正含义。传统开发者依赖编程语言、对代码的深入理解和多年经验。Vibe coder 只需要英语、对技术工作原理的基本理解，以及一个想法。

事实上，大多数 vibe coder 根本不是开发者。约 63% 探索 vibe 工具的用户是非开发者，他们使用 v0 或 Cursor 等工具来简化任务并为自己的问题创建定制解决方案。

这种可访问性带来了新机会，但也带来了新风险，因为许多用户不知道如何保护他们正在创建的东西。

## 构建 Vibe Coding 的护栏

随着越来越多的人采用 vibe coding，安全性必须内置到工具本身中。由于不是每个用户都能接受培训来管理这些，责任就落在了平台上。

Vibe coding 工具必须被设计为处理用户无法掌握的安全考虑。考虑到将信任（和敏感数据）托付给这些产品的人数之多，这一点尤为必要。

对我们来说，这意味着自动干预系统，实时扫描和阻止漏洞，或者具有紧急停止功能，实时保护用户免受不安全部署的影响。

最成功的 vibe coding 平台将是那些将安全性直接构建到其界面中的平台。

![Vibe Coding 安全性](/images/articles/2025-02/vibe-coding-security.png)

## Vibe Coding 的未来

对 vibe coding 的兴趣只会继续增长。

企业可以采用 vibe coding，创建更精简的团队，能够更快、更高效地构建，而无需传统经验。或者，他们可以忽视 vibe coding，坚持传统编程来处理复杂、重要的项目，只在安全和安全性不受威胁的地方使用 AI。

无论如何，转变正在进行中。通过查看我们的[完整 Vibe Coding 现状报告](https://v0.app/vibecoding)，确保你已做好准备。
