## 摘要

Thoughtworks 的多位架构师在这期节目中以 “年度词汇回顾” 的方式再次审视 vibe coding，强调这不是一个简单的技术潮语，而牵涉到组织结构与工程文化的深层变化，关键词包括 software architecture、AI-augmented development、team practices、engineering culture。 播客一开始就强调目前对 vibe coding 的定义存在大量分歧：有的人认为只是把 AI 当高级自动补全，有人则把它等同于全程对话式开发，因此他们花时间给出一个工作级定义，以便讨论后续的实践与风险。 争议焦点在于，主持人们一方面承认在内部项目中确实看到团队通过 vibe coding 大幅缩短交付时间，另一方面也注意到在缺乏设计评审、测试策略和知识沉淀机制时，系统演化会变得异常脆弱，因此节目提出企业应把 vibe coding 纳入架构治理与平台工程框架中，而不是放任其自由生长。​ -
