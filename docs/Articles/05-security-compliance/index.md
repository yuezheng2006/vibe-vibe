# 安全、合规与局限性

很多关于智能体的讨论都在强调能力：能读文件、能跑命令、能调 API、能连续完成长任务。但一旦这些能力真的进入生产环境，问题很快就会换个问法——它会出什么事，以及出事时靠什么边界兜住。

这个栏目虽然篇幅不长，却回答了生产级 Agent 系统最基础的两个问题。第一，威胁会从哪里来：提示注入、工具滥用、权限扩张、数据泄露，以及能力组合后带来的系统性攻击面。第二，当你已经知道这些事迟早会发生，系统边界应该怎样设计，才能既保留能力，又把事故半径控制在可以承受的范围内。

读完这一栏，你获得的不该只是“要注意安全”这种正确废话，而是两种真正可用的判断：一是能识别智能体系统最危险的接口和耦合点，二是能理解不同边界模型分别适合什么场景、挡得住什么问题、又会牺牲什么能力。
