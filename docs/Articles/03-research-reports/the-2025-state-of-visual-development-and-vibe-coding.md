## 摘要

Bubble 的这份报告通过大规模问卷对比“可视化开发（no/low-code）”与“vibe coding”在真实项目中的采用与信心差异，核心关键词包括 visual development vs vibe coding、builder confidence、mission-critical applications、security & compliance。 报告指出，约 25.6% 的受访者会同时使用可视化开发和 vibe coding 平台，但在信心和生产部署比例上差距明显：71.5% 对用可视化开发做关键应用感到有信心，而对 vibe coding 只有 32.5%；实际仅 9% 会用 vibe coding 工具承载业务关键应用，而 65.2% 会用可视化开发完成多数生产应用。 创新点在于把 vibe coding 放在与成熟 no/low-code 工具的直接对比中：报告认为后者在可视化、可审计和安全合规上优势明显，而 vibe coding 如不解决“黑箱代码”和可审计性问题，很难获得企业级信任。 在市场预测部分，Bubble 预计可视化开发在未来几年采用增速将超过 vibe coding（75.7% vs 50% 计划增加使用），并强调安全与合规顾虑会持续限制后者在生产场景的渗透，为业内关于“哪种范式更适合作为默认开发方式”提供了硬数据。
