## 摘要

这篇文章虽然是个人博客，但在后续大量研究报告中被当作对 vibe coding 概念的早期技术性梳理，核心关键词包括 AI-assisted programming、vibe coding、tooling spectrum、context engineering。 作者区分了“普通 AI 辅助编程”和“真正意义上的 vibe coding”：前者是用补全和对话提高效率，后者是尽量通过自然语言描述意图，让 AI 承担大部分实现细节，人类只做方向把控和审阅。 创新点在于提出“谱系视角”：将开发实践从传统手写、轻度 AI 辅助、到全流程 vibe coding 排成一个连续轴，并讨论在哪些环节必须引入规格、测试和审计，才能避免质量失控。 虽然不是正式白皮书，但这篇文章后来被多份行业报告引用，用作区分 hype 与现实的概念锚点，也间接推动了“vibe coding 需要 spec”这一后续研究话题的形成。
