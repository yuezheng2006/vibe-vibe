## 摘要

在这期通讯中，开发者与开源作者 Simon Willison 提出“vibe engineering”概念，试图为社区对 vibe coding 的讨论提供更精确的技术语言，核心关键词包括 vibe engineering、prompt design、system behavior shaping、AI-assisted development。 他认为，如果把“随缘聊天写代码”也叫 vibe coding，就容易掩盖真正需要的工程工作：实际上，工程师要做的是通过 prompt、示例和工具接入来“工程化”系统的 vibe，使其在特定任务上表现稳定可预期。 创新点在于强烈主张为 vibe coding 引入“规范与规格”的视角：文章与他在其他渠道提出的“vibe coding 也需要 spec”观点呼应，鼓励团队像写 API 文档那样，为 AI 助手的角色、约束和期望行为写清楚说明。 虽未给出实验数据，他引用自己在开源项目和客户项目中的经验，指出一旦将“vibe 视为可配置参数，而非玄学”，AI 在代码生成和重构中的可控性会显著提高，也让 vibe coding 更接近可靠的工程实践。
