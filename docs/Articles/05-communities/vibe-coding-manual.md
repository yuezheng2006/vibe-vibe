## 摘要

这篇被社区长期引用的《Vibe Coding Manual》尝试为“如何和 AI 写代码”建立一套约定俗成的行为规范，核心关键词包括 communication preferences、implementation plan、change sizing、AI collaboration workflow 等。 文档通过社区成员共创的方式，把大家在实际用 LLM 写代码时的偏好总结成可复制的指令模板，例如要求模型在大改动前先给出实现计划、对每次修改进行 S/M/L 级别量化、在回复中标注“已完成/待办”等，从而减少“凭感觉乱改”的风险。 创新点在于将“情绪与语气”也纳入规范：作者建议用明确的情绪提示来提高模型对关键任务的重视度，并通过示例说明这种“情绪信号 + 技术约束”的组合更容易让 LLM 遵守安全与审慎原则。 虽然没有正式研究数据，但该手册在回复区获得大量点赞与补充，被不少用户视为从“随缘 vibe coding”迈向“可控协作工作流”的第一步，也成为后续各家工具规则模版的灵感来源。
