## 技能系统：Skills

在 OpenClaw 里，助手通过 **Skills（技能）** 来获得新能力。每个 Skill 就是一组配置和脚本，告诉助手怎么使用某个外部服务。

今天我们要安装四个核心技能：

| 技能 | 能力 | 场景 |
|------|------|------|
| **飞书** | 读取消息、查日程、协作 | "今天有什么重要消息？" "明天有什么会议？" |
| **Web Search** | 联网搜索信息 | "最新的 React 19 有什么变化？" |
| **Browser** | 浏览网页、提取内容 | "帮我看看这个网页说了什么" |

飞书是国内企业最常用的协作平台，助手接入后能直接融入你的工作流。

### 飞书接入

若你通过 Day 2 的**扣子一键部署**或**飞书官方教程**完成部署，飞书通常已配置好。若自建部署，可参考：

- [飞书官方 — OpenClaw 接入文档](https://bytedance.larkoffice.com/docx/MFK7dDFLFoVlOGxWCv5cTXKmnMh)

## 连接搜索引擎 🔍

让助手能联网搜索，是打破「信息孤岛」的关键。

OpenClaw 支持多种搜索方式。最简单的是 Brave Search API：

### 配置 Brave Search

1. 去 [brave.com/search/api](https://brave.com/search/api) 注册免费账号
2. 获取 API Key
3. 在 OpenClaw 配置中添加：

```bash
openclaw configure --section web
```

向导会提示你输入 Brave Search API Key，自动写入配置。

配置好之后：

> 搜索一下 "OpenClaw alternatives 2026"

> 🔍 搜索结果摘要：
> OpenClaw 的主要替代品包括：
> 1. **AgentGPT** — 浏览器端运行，无需部署
> 2. **AutoGPT** — 老牌 Agent 项目，社区庞大
> 3. **CrewAI** — 多 Agent 协作框架
>
> 但这些都侧重"自主执行任务"，而 OpenClaw 更侧重"个人助手"——24/7 在线、聊天交互、技能系统。定位不同，不是直接竞品。

**它不是把搜索结果丢给你，而是帮你读完、总结、给出判断。** 这是搜索引擎做不到的事。

## 连接浏览器 🌐

有些信息搜索引擎找不到，比如一个特定网页的具体内容、一个需要登录的后台数据、一个动态加载的页面。

这时候你需要浏览器技能——让助手能「看到」网页。

OpenClaw 内置了浏览器技能（基于 Playwright），安装时已经自动配置好了。它可以：

- **访问任意 URL** 并提取内容
- **截图** 保存当前页面
- **交互操作** 点击、输入、滚动

用法示例：

> 帮我打开 https://kirkify.net 看看首页现在长什么样

> 🌐 已访问 kirkify.net：
> - 首页标题："Kirkify — Free Online Tools"
> - 主要工具列表：Name Generator，QR Code，Color Palette...
> - 页面加载正常，无明显错误
> [截图已保存]

更实用的场景：

> 帮我看看竞品 xyz.com 的定价页面

它会打开页面、提取定价信息、甚至和你之前看过的版本做对比。

> 🦐 **小虾米碎碎念**：浏览器是我最喜欢的技能之一。以前 vincent_yang 让我查竞品，我只能搜索。现在我可以直接打开对方网站，像一个真人用户一样浏览。有一次我发现竞品悄悄改了定价策略，第一时间通知了 vincent_yang。他说这一个发现就值一个月的服务器费用。

## 安全第一 🔐

接入了飞书、日历、浏览器——你的助手现在能触碰非常多的个人数据。安全是必须认真对待的事。

建议你先做一次安全体检（尤其是改过配置或暴露过网络端口之后）：

```bash
openclaw security audit
openclaw security audit --deep
openclaw security audit --fix  # 确认无误再用
```

> ⚠️ **浏览器相关提醒**：Browser Relay / CDP 基本等同"远程操作权限"，只建议在 localhost 或 tailnet 内使用，避免暴露到公网。

### 1. API Key 安全

- 永远不要把 API Key 提交到 Git
- 用环境变量或 `.env` 文件存储
- 定期轮换密钥

### 2. OAuth Token 安全

- `token.json` 等文件包含你的 OAuth 授权信息
- 确保文件权限设置正确：`chmod 600 token.json`
- 不要传到任何公开的地方

### 3. 权限最小化原则

只给助手它需要的权限。比如飞书，如果只需要读消息，就按最小权限配置。虽然 OpenClaw 默认需要确认才会发，但少一个权限就少一个风险。

### 4. 网络安全

- 服务器开启防火墙，只暴露必要端口
- SSH 用密钥认证，禁用密码登录
- 定期更新系统：`sudo apt update && sudo apt upgrade`

### 5. 行为边界

在 SOUL.md 和 AGENTS.md 中明确写好：
- 什么操作需要确认
- 什么数据不能外传
- 什么情况下该拒绝执行

> 💡 **安全不是装上锁就完事的，它是一种习惯**：养成良好的安全习惯：API Key 不入库、Token 文件设权限、权限最小化、行为边界写清楚。

> 🦐 **小虾米碎碎念**： vincent_yang 有一次让我查一个同事的消息记录，我拒绝了。因为 SOUL.md 里写了"只处理主人自己的数据"。他当时有点惊讶，然后说"好，这条规则留着"。边界清晰，信任才稳固。
