## 选择你的运行环境

首先，你需要一个地方让助手「住」下来。

### 方案 A：扣子（Coze）一键部署（推荐）

**零配置、零运维**，在 [扣子 Coze](https://www.coze.cn) 上一键部署 OpenClaw，几分钟即可上线。

- **优点**：无需自建服务器，24 小时在线，国内访问稳定
- **适合**：想快速上手、不想折腾服务器的用户

👉 **详细教程**：[扣子官方文档 - OpenClaw 一键部署](https://docs.coze.cn/tutorial/openclaw)

### 方案 B：自建云服务器

有运维经验、希望完全自主可控？可以租一台 Linux 服务器（如腾讯云、阿里云、AWS 等）自行部署。

- **优点**：完全掌控，数据在自己手里
- **缺点**：需要一定运维能力

### 方案 C：你正在用的电脑

想先体验一下再决定？直接在当前电脑上跑。

- **优点**：零门槛，立刻开始
- **缺点**：关机就没了，适合试玩

> 🦐 **小虾米建议**：新手优先选方案 A，扣子一键部署最快。确认好用后再考虑自建服务器也不迟。

## 准备工作

**方案 A（扣子一键部署）**：直接按 [扣子 OpenClaw 教程](https://docs.coze.cn/tutorial/openclaw) 操作即可，文档会引导你完成部署和飞书接入，无需额外准备。

**方案 B/C（自建部署）**：需要准备：

1. **AI 模型访问** — 有 Claude 订阅可在向导里 OAuth 登录；或去 [console.anthropic.com](https://console.anthropic.com) 创建 API Key
2. **飞书凭证** — 在 [飞书开放平台](https://open.feishu.cn/app) 创建应用，获取 App ID 和 App Secret。飞书接入步骤可参考：[飞书官方文档 - OpenClaw 接入](https://bytedance.larkoffice.com/docx/MFK7dDFLFoVlOGxWCv5cTXKmnMh)、[腾讯云保姆级教程](https://cloud.tencent.com/developer/article/2625073)

> 💡 **为什么推荐飞书？** 国内企业使用飞书的比例很高，团队协作、审批、日历都在飞书里，助手接入飞书后能直接融入你的工作流。若需使用 Telegram、WhatsApp、钉钉、企业微信等其他 IM，请查看 [OpenClaw 官方文档 - Channels](https://docs.openclaw.ai) 的配置说明。

## 部署与配置

### 方案 A：扣子一键部署（推荐）

打开 [扣子 OpenClaw 教程](https://docs.coze.cn/tutorial/openclaw)，按文档步骤操作。扣子会引导你完成：
- 部署 OpenClaw 实例
- 连接飞书
- 配置 AI 模型

几分钟即可在飞书里和助手对话。

### 方案 B/C：自建部署

在终端执行：

```bash
curl -fsSL https://openclaw.ai/install.sh | bash
```

安装脚本会进入交互式配置向导。在「配置聊天渠道」步骤选择 **飞书**，按提示填入飞书应用的 App ID、App Secret 等。具体字段含义和飞书应用创建步骤，请参考 [飞书官方文档](https://bytedance.larkoffice.com/docx/MFK7dDFLFoVlOGxWCv5cTXKmnMh)、[腾讯云保姆级教程](https://cloud.tencent.com/developer/article/2625073) 或 [OpenClaw 官方文档](https://docs.openclaw.ai)。

其他渠道（Telegram、钉钉等）的配置请查阅 [OpenClaw 官方文档](https://docs.openclaw.ai)。

## 日常管理命令

装好之后，这些命令你会经常用到：

```bash
openclaw status          # 查看整体状态
openclaw gateway status  # 查看 Gateway 运行状态
openclaw health          # 健康检查
openclaw configure       # 重新配置（修改模型、频道等）
openclaw daemon restart  # 重启后台服务
openclaw daemon logs     # 查看运行日志
```

## 常见问题排查

### ❓ 安装脚本报错

**Node.js 版本太低**：OpenClaw 需要 Node.js 22+。检查版本：

```bash
node -v
```

如果版本不够，安装脚本通常会自动帮你装。如果没有，手动升级：

```bash
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.0/install.sh | bash
nvm install 22
```

### ❓ 飞书没有回复

- 确认飞书应用凭证（App ID、App Secret）正确
- 确认应用已发布、权限已开通
- 检查日志：`openclaw daemon logs`
- 确认 API Key 有效且有余额

若使用其他 IM（Telegram、钉钉等）遇到问题，请查阅 [OpenClaw 官方文档](https://docs.openclaw.ai) 对应渠道的配置说明。

### ❓ 想重新配置？

随时可以重新运行向导：

```bash
openclaw onboard
```

或者只改某一部分：

```bash
openclaw configure
```
