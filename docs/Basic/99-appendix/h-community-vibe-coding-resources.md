# H. 社区 Vibe Coding 资源

## 这一页是干什么的

本书主线是：**按章节把第一个作品做成、再把全栈与上线走通**。除此之外，中文社区里还有一些仓库把「Vibe Coding」整理成**可复制的提示词、Skills 与文档结构**。这一页只负责**指路和对照**，不替代正文。

## vibe-coding-cn（推荐对照）

仓库地址：[github.com/2025Emma/vibe-coding-cn](https://github.com/2025Emma/vibe-coding-cn)

它是社区维护的 **Vibe Coding 指南** 型项目（MIT 许可，README 含多语言入口）。你可以把它理解成：**方法论 + 资产库**——大量内容以「提示词 / 文档 / Skills」形式组织，适合在本书主线之外**按需拷贝、改造、接入你自己的工作流**。

**站内已收录该仓库根目录 README 的全文转载**（便于离线式通读）：见 [Vibe Coding 指南（社区原文）](/VibeCodingCN/) → [README 全文](/VibeCodingCN/02-readme-full)。

### 和本书的互补关系

| 维度 | 本书（VibeVibe） | vibe-coding-cn |
|------|------------------|----------------|
| 主线 | 从零到一作品 + 进阶全栈与工程化 | 强调规划、上下文固定与可审计流程 |
| 形态 | 教程章节 + 实践案例 | 仓库内分类的 prompts、documents、skills 等 |
| 最适合 | 跟着做完整条学习路径 | 想扩充「话术模板、系统提示、目录约定」时检索 |

### 你可以从中重点补什么

1. **文档化工作流**：用固定目录（例如 README 中描述的 `memory-bank` 思路）承载产品/技术意图、实施计划、进度与架构笔记，让会话切换后仍能接续上下文。
2. **提示词分层**：按「系统 / 编程 / 用户」等类别管理提示词，减少临时拼句子带来的不一致。
3. **Skills 与工具链**：把常用能力沉淀成可加载的 Skill；仓库内还有围绕提示词表格与 Markdown 互转等自动化思路，可按需借鉴。
4. **清单式原则**：对方用「道、法、术、器」等框架做自检——适合当**速查表**，与本书各章细节不必一一句对齐。

### 使用方式建议

- **先做本书作业**：避免在多个体系之间来回换骨架，反而打断节奏。
- **再按需「偷」模板**：缺哪类提示词或目录约定，再去对方仓库对应路径找参考，改写成适合自己项目的版本。
- **注意时效**：模型名、工具 CLI、订阅形态变化快；任何仓库里的「当前最强模型」列表都要结合官方文档与你自己的预算再判断。
