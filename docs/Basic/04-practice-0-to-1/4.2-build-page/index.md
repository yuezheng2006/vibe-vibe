# 4.2 从访客视角决定你要补什么内容

页面一旦开始变得像作品，很多人就会自然进入下一种冲动：继续加，继续补，继续往里塞更多信息。问题是，内容不是越多越好。对个人主页来说，更重要的是相关。别人来到这里，通常不是想把你的一生读完，而是想快速判断你是谁、做过什么、值不值得继续联系或交流。真正决定你该补什么的，不是“我还能写多少”，而是“访客最想先知道什么”。

所以这一节最重要的动作，是从自己切换到访客。你可以直接想三类人：朋友、HR、潜在合作伙伴。他们进入你的主页时，想看到的东西并不完全一样，但会有交集。朋友想快速知道你最近在做什么，HR 想快速知道你做过什么、能力大概在哪，潜在合作伙伴则更关心你到底擅长什么、怎么联系你。只要你先从这个角度看页面，就会很容易发现：有些信息其实没那么重要，有些信息却应该更早出现。

## 先问一个比“我还想加什么”更有用的问题

你完全可以先写下一句很简单的话：

> 如果别人只花 30 秒看我的主页，他最应该记住什么？

这个问题会直接帮你筛掉很多其实不急着加的东西。因为它逼你从“我还能写什么”切换到“别人最该先看到什么”。

## 三类访客，通常会关心什么

| 访客 | 他们通常最想先知道什么 |
|------|------------------------|
| 朋友 | 你最近在做什么，有没有新作品或新方向 |
| HR / 面试官 | 你是谁，做过什么，能力大概落在哪 |
| 潜在合作伙伴 | 你擅长什么，值不值得继续交流，怎么联系 |

你不需要把三类人的需求都分别做一套页面，但可以从它们的交集里找优先级。很多时候，交集其实很稳定：你是谁、你做过什么、最近在做什么、怎么联系你。

## 内容不是越多越好，而是越相关越好

这也是为什么第 4 章不鼓励你“见模块就加”。你现在要做的，不是往主页里堆一个百科全书，而是判断哪些内容最能帮助别人快速理解你。

一个很实用的判断标准是：这个模块加上去之后，能不能让访客更快回答下面至少一个问题？

- 这个人是谁？
- 他最近在做什么？
- 他做过哪些值得看的东西？
- 如果我想继续联系，应该怎么做？

如果一个模块不能明显帮助回答这些问题，它就很可能没有现在想象中那么重要。
