# 4.4 本章小结：主页更完整，也更可回退

如果第 3 章解决的是“它看起来更像作品了”，那第 4 章解决的就是“它终于不再只是一个好看的空壳”。你已经开始从访客视角看页面，知道别人最需要看到什么，也开始用更克制的方式补内容、补引导。与此同时，你还给项目装上了一个最基础的后悔药，让自己在继续改的时候不至于一直提心吊胆。

这两个变化合在一起，会让你的作品进入一个新阶段。别人进来之后，不只是觉得它比以前更顺眼了，还会更容易理解你是谁、做过什么、接下来可以点哪里、问什么。你自己也不会再因为“万一改坏了怎么办”而一直缩手缩脚。到这里为止，页面已经不只是一个 demo，而更像一个开始可以被认真看待的作品。

## 回头看，这一章真正带来了什么

你已经会做最基础的存档与回退，也已经开始用访客视角筛选内容，而不是单纯往页面里堆信息。你补上了 2 到 3 个最有价值的内容模块，也让数字分身入口比之前更容易被真正用起来。

## 验收清单

- [ ] 我已经知道怎么给项目做最小存档，并在必要时退回来
- [ ] 页面内容比之前更完整，但没有失控地越加越多
- [ ] 访客更容易理解我是谁、做过什么、怎么联系我
- [ ] 数字分身入口比之前更容易被注意到，也更容易被实际使用

## 下一步

接下来，重点会从“页面像不像作品”继续往里走，进入“数字分身像不像你”。第 5 章会开始处理更核心的一层：它为什么会这样回答，怎么让它更像你本人，以及遇到不回复、答偏、边界不清时，你该怎么排查。
