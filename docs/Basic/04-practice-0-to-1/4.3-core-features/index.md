# 4.3 只添加 2-3 个最有价值的内容模块，并让数字分身更容易被使用

有了访客视角之后，下一步就不是“加所有东西”，而是只加 2 到 3 个最有价值的模块。这里最容易犯的错误，是把主页想成一个越完整越好的仓库。其实基础版更鼓励你做减法后的完整：不是所有模块都要有，而是先加那些最能帮助别人理解你、联系你、继续和你互动的内容。

## 哪些模块通常最值得优先考虑

对大多数个人主页来说，最值得优先考虑的通常就那么几类：作品展示区、联系方式、经历时间线、FAQ。你不一定全都要。很多时候，三选二或者四选二就已经足够。

你可以先这样判断：

| 模块 | 它解决什么问题 |
|------|----------------|
| 作品展示区 | 让别人更快知道你做过什么 |
| 联系方式 | 让“感兴趣”可以变成“能联系到” |
| 经历 / 时间线 | 帮别人快速看懂你的背景脉络 |
| FAQ | 回答高频问题，降低访客开口成本 |

关键不在于模块名字，而在于每加一个模块之前，都先问自己：它是不是在帮助访客更快理解我，而不是只是让我感觉“页面更满了”。只要这个问题能答得出来，这个模块就更可能值得加。

## 一个很稳的做法：一次只加 2 到 3 个

这一章不建议你把所有能想到的模块都一次补齐。更稳的做法，是只选 2 到 3 个最有价值的模块。

比如：

- 如果你现在最缺“别人看不懂我做过什么”，优先补作品展示区
- 如果你现在最缺“别人想继续交流却不知道怎么找我”，优先补联系方式
- 如果你现在最缺“别人不知道该问什么”，优先补 FAQ 或聊天区引导

这种节奏有两个好处：一是页面不会一下变得很杂，二是你更容易判断，到底是哪一轮修改真正让它变完整了。

## 数字分身入口，不该只是“有个聊天框”

与此同时，别忘了数字分身入口本身也需要被设计。很多页面的问题不是“没有聊天功能”，而是别人根本不知道该怎么用它。一个会聊天的组件，不等于一个容易被使用的入口。访客不知道问什么，不知道这里能回答什么，或者根本没注意到它，那这个功能的价值就很难真正出现。

所以这一节也很适合顺手补一点引导，比如：

- 一两句更清楚的提示文案
- 几个快捷问题按钮
- 一个告诉别人“你可以在这里问我最近在做什么、我有哪些作品、怎么联系我”的入口说明

这类引导不花哨，但特别重要。因为它会把“有一个聊天框”推进成“这个聊天框真的有人会用”。而这恰恰是作品体验开始从 demo 往真实使用靠近的地方。

## 操作节奏也可以设计得更稳

在操作层面上，你也可以把节奏控制得更稳一点：每加一类模块前先做一次存档，改完再看一次效果。这样你既不会因为改动变大而慌，也能比较清楚地判断，到底是哪一轮调整真正让页面变得更完整了。

如果你不知道怎么让 AI 帮你加模块，可以直接说：

```text
请帮我给个人主页补一个作品展示区和一个联系方式区。
目标是让访客更快知道我做过什么，以及怎么联系我。
请保持现在的整体风格，不要一次新增太多模块。
加完之后，再帮我检查聊天入口是否足够明显。
```

这种说法很适合现在这个阶段：它既说清了目标，也把范围收住了。
