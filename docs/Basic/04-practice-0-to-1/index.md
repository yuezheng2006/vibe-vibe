# 第 4 章：让主页更完整 —— 内容、引导与基础存档

第 3 章让你的首页开始像一个作品了，但“像作品”和“足够完整”之间还差一步。很多页面看起来不难看，问题却在于别人进来之后还是不知道该看什么、问什么、怎么联系你。与此同时，随着改动越来越多，你也会开始碰到另一个现实问题：万一哪次改崩了怎么办？所以第 4 章要同时处理两件事：一边让主页变得更完整、更可用，一边在复杂改动开始之前，先给它装上一个最小的后悔药。

这章的重点不是堆更多功能，而是用更克制的方式补上真正有价值的部分。你会开始从访客视角看页面，判断别人最想知道什么；你会补 2 到 3 个最值得先加的模块，而不是无限往页面里塞内容；你也会把数字分身入口做得更容易被使用，而不是只停留在“有一个聊天框”。与此同时，你会第一次接触最基础的 Git 闭环：存一次、看历史、退回来。它在基础版里不是协作工具，而是后悔药。

第 4 章很像一次收口。前面你已经让页面能跑、也更像样了；这一章要做的，是让它开始更完整，也更像一个别人真的会停下来看看、甚至愿意继续互动的东西。

## 这一章要把作品推进到哪里

如果说前面三章更像在搭框架、定气质，那这一章会把作品往“真的可以给别人认真看一眼”的方向推进。你会开始补上内容，也开始补上使用体验。到了这一步，一个页面就不只是“能打开、能聊天”，而是更像一个有人真的会停留、会理解、会继续互动的东西。

更具体一点说，你会拿到两样东西：一是更完整的主页，二是更放心的修改心态。前者来自内容和引导，后者来自最小的存档与回退能力。
