# 2.3 完成一次最小修改，并学会 3 类高频提问

项目在本地跑起来之后，最值得立刻做的事，不是挑战一个大改动，而是先完成一次很小的修改。越小越好。改一句欢迎语，改一段自我介绍，改一个按钮文案，甚至只是把某个标题换成更像你自己的说法，都可以。这里最重要的不是结果有多惊艳，而是你要亲手经历一次完整闭环：我提出修改，AI 帮我改，本地页面真的发生变化。

这个闭环对新手特别重要，因为它会把“本地开发”从抽象概念变成身体记忆。你会第一次明确知道，原来所谓在本地继续做项目，不是突然开始手写大量代码，而是可以从一个非常小的动作开始。你只要看见页面随着你的意图发生变化，后面所有更复杂的修改，就都有了落脚点。

## 第一次修改，越小越好

这里推荐你改的不是结构，不是功能，而是一处非常容易看见变化的内容。比如：

- 把首页的一句话介绍改得更像你
- 把按钮文案从默认说法改成更自然的说法
- 把聊天区引导语改成“你可以问我最近在做什么”这种更具体的句子

为什么要这么小？因为你现在练的不是产出规模，而是流程感。只要你已经经历过“提出修改 -> AI 改动 -> 本地页面变化 -> 我确认结果”这一轮，后面再做更大的事情，心里会稳很多。

## 一个很够用的第一次任务

你可以直接选下面这种级别的动作：

```text
请帮我把首页的一句话介绍改得更像我本人。
目标文案是：一个正在学习用 AI 做产品的内容策划。
请告诉我这句文案在哪个文件里，并只修改这一处，不要动其他布局和样式。
```

这句话的价值很高，因为它同时具备三样东西：目标、范围、约束。对 AI 来说，这比一句“帮我改一下首页”清楚得多；对你自己来说，它也在训练一种很关键的习惯：不是模糊地下指令，而是把动作说清楚。

## 这三类问题，后面会一直用

到了这里，也很适合顺手建立三类以后会高频重复使用的提问方式。

### 1. 理解类：这个项目结构在做什么？

这类问题帮你快速认识地图，避免你每次都像第一次进来一样陌生。

```text
请用人话解释这个项目目前的结构。
告诉我主页入口、聊天相关代码、样式相关文件分别大概在哪。
不要展开太深。
```

### 2. 定位类：我想改 X，代码在哪？

这类问题帮你在真正动手前，先找到入口。

```text
我想修改首页首屏的一句话介绍和主按钮文案。
请先告诉我这两处分别在哪个文件里，再给出修改建议。
```

### 3. 约束类：只改这一处，不要动其他地方

这类问题帮你把范围收住，尤其适合你还不想让 AI 一次动太多的时候。

```text
请只修改聊天框的引导语，不要改布局，不要新增模块，也不要调整配色。
```

你会发现，这三类话术几乎贯穿后面所有章节。它们不花哨，但特别实用，因为它们直接对应你作为独立 vibe coder 最常见的三个动作：理解、定位、约束。

## 如果你想把这一节做扎实一点

可以连续完成两件小事：先改一句文案，再改一个按钮或一个区块标题。只要这两轮都能顺利跑通，你就会开始明显感觉到，自己已经不是在“看平台表演”，而是在真的驱动一个项目往前走。

到这里，你就已经拥有了后面持续修改时最常用的一组基本动作。
