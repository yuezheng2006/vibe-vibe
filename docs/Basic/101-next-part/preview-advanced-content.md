# 进阶版核心内容详解

如果说基础版是一条“先做出来”的主线，那么进阶版更像一张“哪里不够稳，就去哪里补”的能力地图。它不是要求你从头到尾线性学完，而是帮你在继续做项目时，把那些已经碰到过边界的部分系统补齐。

下面这些，是进阶版最核心的几块内容。

## 1. 环境与工具：把工作台搭得更稳

当你已经把项目带回本地，下一步很自然会碰到环境、终端、依赖、端口、包管理这些问题。基础版里我们只讲到“够你继续推进”的程度，进阶版会把这一块系统补上。

- [第一章：环境搭建](/Advanced/01-environment-setup/)

适合你的时机：你已经能在 AI 帮助下把项目跑起来，但还不太清楚终端、依赖、端口、配置文件这些东西到底在做什么。

## 2. AI 工作流与调试：让协作更稳，而不是更玄学

基础版已经反复在用“卡住就问 AI”的思路，但当项目越来越复杂时，你会开始需要更系统的工作流：什么时候先让 AI 规划，怎么写规则，怎么排错，怎么利用 Skills 或 MCP，怎么让多轮协作不发散。

- [第二章：AI 使用说明书](/Advanced/02-ai-tuning-guide/)

适合你的时机：你已经不是不会用 AI，而是开始想让 AI 更稳定、更高效、更可控。

## 3. 产品思维与 PRD：把想法讲得更完整

基础版第 1 章里，你已经写过第一版需求；第 4 章里，你也开始从访客视角思考内容和体验。进阶版会把这件事继续往前推，讲得更系统：怎么确认问题，怎么和 AI 讨论方案，怎么把需求写成更稳的文档。

- [第三章：产品思维与文档驱动](/Advanced/03-prd-doc-driven/)

适合你的时机：你开始发现，项目做不稳，往往不是不会写代码，而是前面的想法、范围和需求没讲清。

## 4. UI / UX：从“改好看一点”走到更像作品

基础版第 3 章已经让你开始学会表达风格、调整首屏和处理最小体验问题。进阶版会把这一块补得更完整：设计工具怎么用，组件库怎么选，动画什么时候有价值，复杂界面如何继续推进。

- [第五章：好看好用的界面](/Advanced/05-ui-ux/)

适合你的时机：你已经能看出页面问题，但想让自己的表达更具体，也想知道哪些工具和方法更适合继续往上走。

## 5. API 与后端：理解聊天、数据和交互背后的链路

基础版第 5 章已经用“数字分身说明书”和“API 是桥”给了你一个够用心智模型。进阶版会继续把这条链路讲清楚：接口是什么，前后端怎么配合，为什么有些问题像前端问题，实则是 API 或数据层问题。

- [第七章：连接前后端](/Advanced/07-backend-api/)

适合你的时机：你已经不满足于“它能跑”，而想知道请求到底怎么发出去、怎么返回来、哪里会出问题。

## 6. 安全与密钥：把底线补扎实

基础版只要求你建立最低限度的安全意识，比如不要把 API Key 写进代码仓库，上线前要检查环境变量。进阶版会把这一块系统补上：密钥、环境变量、认证、路由保护、常见安全风险。

- [第八章：谁能访问我的数据](/Advanced/08-auth-security/)

适合你的时机：你开始接真实服务、真实用户、真实数据，不想再只靠“我大概知道要小心”来撑着。

## 7. Git 协作：从“后悔药”到正式工作流

基础版第 4 章里的 Git，只承担最小存档与回退。进阶版会继续往前讲：远程仓库、分支、Pull Request、协作节奏，以及为什么这些东西一旦项目变大就会变得重要。

- [第十一章：协作开发](/Advanced/11-git-collaboration/)

适合你的时机：你已经会存档和回退，但开始想把项目交给更正式的版本管理方式，或者和别人一起协作。

## 8. 部署、CI/CD 与正式发布：让上线不只是“一次成功”

基础版第 6 章已经带你走过了最小发布闭环。进阶版会继续把部署讲完整：不同平台的差异，自动化部署，成本与运维，为什么有时本地能跑、线上却不稳。

- [第十二章：无服务器自动部署](/Advanced/12-serverless-deploy-cicd/)

适合你的时机：你已经有了公网链接，但想把上线这件事做得更稳、更可持续，而不是每次都像临时冲刺。

## 9. 域名、DNS 与正式接入：让作品更像真正的产品

很多人第一次上线之后，很快就会问下一个问题：我能不能用自己的域名？备案是什么？DNS 到底在干嘛？进阶版会把这一块补上，让你从“有一个平台链接”走到“更正式地对外访问”。

- [第十三章：域名解析与接入](/Advanced/13-domain-dns/)

适合你的时机：你准备把作品长期放出来，或者希望它在外部看起来更完整、更正式。

## 不止这些，但先抓这些最值回票价的块

除了上面这些核心块，进阶版站点里还有开发基础、数据库、测试自动化、本地公网访问、VPS、SEO、用户反馈与迭代等内容。你不需要一次全吞下去，先抓和你当前项目最相关的块，就已经很值回票价。

## 最推荐的进入方式

如果你不知道该从哪里开始，最稳的做法是反过来问自己：**我现在卡住的第一个边界是什么？**

- 卡在本地环境，就先去看环境与工具
- 卡在调试和协作，就先去看 AI 工作流
- 卡在页面质量，就先去看 UI / UX
- 卡在聊天接口和真实能力，就先去看 API 与安全
- 卡在发布、部署、域名，就先去看部署与接入

这样进入进阶版，你会更容易把学到的东西立刻用回当前项目里。
