# 1.4 本章小结：第一轮验收与下一轮优化清单

## 先停一下，别把第 1 章拖长

到了这一步，很多人会自然进入一个陷阱：

> “既然已经做出来了，那我再多改一点，再加一点，再调一点……”

这样很容易把第 1 章拖成一个没有结束点的过程。

所以这一节要做的，就是把完成标准定下来：什么情况下，这一章就算过关？

## 过关线在哪里

第 1 章的过关线其实不复杂。别人点开预览页时，至少应该看到一个能正常打开的完整页面，而不是空白或明显报错；页面里要能清楚表达“你是谁”，至少包括名字、一句话介绍，以及一眼能看出的个人定位；数字分身入口也应该已经出现，哪怕它还不够聪明，也得让人知道这里可以提问，并完成一轮基础对话；手机端则不用追求精致，但至少要保证文字能看、区块不严重重叠、主要按钮还能点。

## 一份直接可用的验收清单

- [ ] 页面能正常打开
- [ ] 我的名字和一句话介绍清楚显示
- [ ] 页面看起来已经像“我的个人主页”而不是完全空白模板
- [ ] 聊天入口存在，并且能完成基础提问
- [ ] 手机端至少可以正常浏览

## 一个很实用的检查动作

不要只用“作者视角”看页面。你可以假装自己是第一次访问的朋友，问自己三个问题：我能不能一眼知道这是谁；我知不知道应该点哪里、问什么；这个页面现在值不值得继续迭代。如果这三个问题大体都能回答“可以”，这个版本就已经过关了。

## 记录下一轮优化清单

第一个版本跑出来以后，你一定已经注意到一些地方还可以更好。这很正常，而且很有价值。因为后面的第 2 章到第 5 章，并不是凭空多讲几章内容，而是在继续解决你现在已经看见的问题。

推荐按这四类来记：

| 类别 | 你可以怎么记 |
|------|---------------|
| 界面问题 | 颜色不喜欢、排版不顺、聊天框贴边、移动端不舒服 |
| 内容问题 | 自我介绍太空、名字信息有误、作品区还没有、联系信息不完整 |
| 数字分身问题 | 回答太机械、Q1/Q2/Q3 还是占位、还没接真实模型、不够像我 |
| 体验问题 | 访客不知道聊什么、按钮文案不清楚、链接不能跳、入口不明显 |

一个可以直接复制的模板：

```text
我的下一轮优化清单

【界面】
- ______
- ______

【内容】
- ______
- ______

【数字分身】
- ______
- ______

【体验】
- ______
- ______
```

比如如果你拿到的第一页结果和上一节截图类似，你完全可以先这样记：

```text
我的下一轮优化清单

【界面】
- 聊天框离右边缘太近，视觉上有点挤
- 首屏和聊天区还不够协调

【内容】
- 名字 / 中文名显示不对，要改正
- 某些介绍文案还不够像我

【数字分身】
- Q1 / Q2 / Q3 还是演示按钮，不是真实问答
- 聊天区要确认有没有接上真实模型

【体验】
- 底部链接还不能跳转
- 页面里有些按钮看起来能点，但实际上没动作
```

如果你想再往前走一点，也可以顺手加一个“优先级”：

| 问题 | 优先级 | 可能在哪一章解决 |
|------|--------|------------------|
| 首页还像默认模板 | 高 | 第 3 章 |
| 页面内容太少 | 高 | 第 4 章 |
| 访客不知道聊什么 | 高 | 第 4 章 |
| 数字分身不像我 | 高 | 第 5 章 |
| 想让别人稳定访问 | 中 | 第 6 章 |

## 回头看，这一章你已经拿到了什么

你准备了第一批真实素材，也把模糊想法变成了可执行需求；你用 AI 平台生成了一个 `v1` 原型，完成了几轮正常迭代，还做了第一轮验收，并记下了后面几章会继续处理的优化清单。

## 验收清单

- [ ] 我已经有一个能预览的个人主页 + 数字分身 `v1`
- [ ] 我知道这个版本哪里还不够好
- [ ] 我已经把这些问题记成了后续可继续推进的清单

## 下一步

接下来，我们会做一件很关键的事：把这个还停留在平台里的作品，带回你自己的电脑。只有它进了你自己的工作台，你才算真正开始拥有它、管理它、持续修改它。
