# 1.1 为什么从这一章开始，以及先准备什么

## 为什么从这一章开始

很多人一开始就陷入“我是不是应该先学会更多概念”的焦虑。

但对零基础读者来说，最有效的起点通常不是再多读一篇解释，而是先完成第一次正反馈：

> 原来我不用先学完整套语法，也真的能把一个页面做出来。

所以第 1 章最重要的，不是“把所有方法都学会”，而是先把你的第一个作品做出来。

这里的任务不是学会完整提示词理论，也不是一次把作品做到完美，而是先做出一个能预览、能展示、能简单聊天的版本。

## 先别急着打开工具

很多新手一上来就直接对 AI 说：“帮我做一个个人主页，再加一个聊天机器人。”

结果当然也可能生成出一个页面，但它往往很空、很泛、很像模板站。不是因为 AI “不会做”，而是因为你还没给它足够真实的材料。

数字分身尤其如此。

如果你自己都还没整理出“我是谁、我想让别人看到什么、别人最可能问什么”，那它也很难替你回答得自然。

## 这一步真正要做什么

在正式生成之前，你先准备一批 **够用的第一版素材** 就可以。

注意，是“够用”，不是“完美”。这一节的任务还不是写 prompt，而是先把你手里真正有的东西收拢出来。下一节我们才会把这些素材装进一个低门槛模板里。

## 先收集事实，再考虑怎么说

很多人一上来就试图把自己写成一段很完整、很像宣传文案的话，结果反而越写越空。更稳的做法，是先记事实，再记表达。

你先不用追求句子漂亮，只要把下面几类信息记出来就很够用：

| 类型 | 你先记到什么程度 |
|------|------------------|
| 基本信息 | 名字、头像、一句话介绍 |
| 当前状态 | 你最近主要在做什么、最想让别人记住什么 |
| 兴趣与特点 | 2-4 个兴趣点、一个比较有记忆点的个人特点 |
| 访客视角 | 可能来访问你主页的人是谁 |
| 常见问题 | 别人最可能问你的 3 个问题 |

这些内容现在可以先写成很散的草稿，不需要已经像一段完整 prompt。

## 什么样的素材更有用

这里最值得记住的一点是：**具体比抽象更有用，真实比好听更有用。**

比如下面这两种写法，后者通常就更容易让 AI 做出像你的东西：

| 太空的写法 | 更有用的写法 |
|------------|--------------|
| 我是一个有热情的人 | 我最近在整理自己的作品，也在尝试用 AI 做个人项目 |
| 我喜欢很多东西 | 我比较关注 AI 应用、写作和旅行 |
| 我想做一个很棒的主页 | 我想做一个能展示我是谁、也能回答常见问题的个人主页 |
| 希望聊天机器人聪明一点 | 希望数字分身能回答“你最近在做什么”“你有哪些作品”“怎么联系你” |

你会发现，真正有用的素材，不是“文采更好”，而是信息更落地。

## 为什么这批素材这么重要

因为它们会直接决定两个结果。

### 1. 页面是不是像“你自己的页面”

如果你给的信息太空，AI 很容易生成一个看起来没什么问题、但谁都可以套用的默认页面。

### 2. 数字分身是不是像“在替你说话”

数字分身不是百科全书。它最重要的工作，是根据你提供的事实回答问题，用接近你的方式介绍你自己，并在不知道时诚实地收住，而不是乱编。

所以这一步其实是在给后面的页面和数字分身打地基。

## 不需要现在就准备什么

你现在 **不需要** 一份完美简历，也不需要全部作品素材、很复杂的人设文档，或者一套特别完整的个人品牌故事。

基础版的第一章只要求你先准备 **第一版够用材料**。后面第 4 章、第 5 章还会继续把内容和风格补上去。

接下来，我们把这批素材放进一个低门槛的填空式模板里，让这些散的事实先落成一版能执行的需求。
