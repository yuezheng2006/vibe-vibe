# 3.3 用更清晰的话告诉 AI 你想怎么改，并兼顾好不好用

到了这一步，真正拉开差距的，往往已经不是“你知不知道更多设计词”，而是你会不会把当前问题、目标感觉和修改边界一起说清楚。很多无效修改都卡在这里：你只说“改好看一点”，AI 当然也会给你一个结果，但这个结果很容易只是“变了”，而不是“变对了”。

## 更有效的说法，通常有三层

先说现在的问题是什么，比如首屏信息太散、聊天入口不够明显、颜色有点像默认模板；再说你想改成什么感觉，比如更简洁、更多留白、更像一个认真做过的个人作品；最后再把边界说出来，比如先不要新增模块，只改首页首屏和整体配色，保留现在的内容结构。只要这三层同时在，AI 的改动就会稳定很多。

你可以把它记成一个很够用的句式：

```text
现在的问题是什么
-> 我想改成什么感觉
-> 这次只动哪些地方，不动哪些地方
```

比如：

```text
现在这个页面首屏太像默认模板，信息有点散，聊天入口也不够明显。
我希望它更简洁、更有层次，更像一个认真做过的个人主页。
请先只调整首屏排版、颜色和间距，不要新增模块，也不要改聊天逻辑。
```

这种表达没有堆术语，但方向已经很清楚了。

## 你也可以给 AI 一个参考物

如果你脑子里有画面，但说不出那么多词，也没关系。你完全可以给 AI 一个参考物，帮助它更快对齐感觉。参考物可以是：

- 一张你现在页面的截图
- 一个你觉得风格不错的网站
- 一句很短的风格描述

比如：

```text
我想让它更接近“简洁、留白多、信息层次清楚”的感觉，
不要太像简历站，也不要太像炫技 landing page。
```

参考物不是为了让 AI 照抄，而是为了帮助它理解你要往哪里靠。

## 好看之外，还要顺手建立一点“好不好用”的意识

这里也很适合建立一点最小体验意识。界面不只是“好不好看”，也包括“好不好用”。一个页面也许颜色不错、排版也不乱，但如果访客一眼看不懂你是谁，不知道该点哪里，不知道聊天区是干什么的，那它就还没有真正进入“作品”状态。

所以当你让 AI 修改界面时，也可以顺手把这种体验要求说进去：

- 首屏要一眼知道我是谁
- 聊天入口要更明显
- 手机端要保证信息可读
- 按钮要有基本反馈

这些要求不复杂，但特别实用。它们会把“改界面”从单纯好看，往“别人真的会用”推进一步。

## 一个可以直接复用的改界面模板

```text
现在这个页面的主要问题是：______

我希望它改成这样的感觉：______

这次请你只修改这些地方：______
先不要动这些部分：______

另外请兼顾这些最基本的体验：
- 首屏一眼能看出我是谁
- 聊天入口更明显
- 手机端文字可读
- 不要让页面显得更乱
```

写到这个程度，就已经比“帮我改好看一点”强很多了。
