# 3.1 为什么先改界面，以及先选一个清晰的风格方向

很多人会觉得，既然项目已经能跑了，是不是应该先去补功能、学原理、接更多能力。也可以，但对基础版来说，先改界面往往更划算。因为界面是最直观的地方。一个页面是不是像模板，是不是有层次，是不是像一个你愿意发给别人的作品，几乎不用解释，自己和别人都能立刻看出来。它特别适合作为你和 AI 继续协作的下一块练习场。

这里真正练的，其实不是“设计能力”本身，而是表达能力。你脑子里其实已经会有一些模糊判断：这个页面太散了，颜色不太像我，首屏没重点，聊天入口不够明显。问题通常不是你完全没有感觉，而是你还不太会把这种感觉说清楚。第 3 章就是拿最直观的一层，帮你练这件事。

## 为什么界面特别适合作为下一步

因为它反馈最快。你去改 API、改复杂逻辑、改数据结构，变化往往要经过一层理解，才能判断值不值；但界面不同。你调完首屏排版，页面是不是更顺眼，立刻就能看出来；你把聊天入口做明显了，别人会不会注意到，也几乎马上能判断。

对新手来说，这种快速反馈很重要。它会让你更容易建立信心：原来我不是只能说一个很模糊的“改好看一点”，我真的可以一点点把页面往自己想要的方向推过去。

## 风格方向不要多，先求统一

在开始改之前，一个很重要的动作是先选方向，而不是一口气什么都想要。很多页面之所以越改越乱，不是因为改得不够多，而是因为方向不稳定。你一会儿想要简约专业，一会儿又想要温暖友好，再一会儿又想加一点科技感，最后就容易变成三种风格同时存在。对基础版来说，风格不需要多，只要统一。你完全可以先选一个你觉得最接近自己的方向，然后先把这个方向立住。

你可以先从下面三种里选一个最贴近自己的：

| 方向 | 更像什么感觉 | 适合什么场景 |
|------|--------------|----------------|
| 简约专业 | 克制、清楚、信息优先 | 想让别人快速看懂你是谁、做什么 |
| 温暖友好 | 有人味、愿意交流、距离感更低 | 想强调表达、内容、沟通感 |
| 轻微科技感 | 清爽、数字感、稍微有未来感 | 本来就在做 AI、产品、技术相关内容 |

没有哪一个绝对更好，关键在于它是不是统一，以及是不是贴近你想让别人感受到的第一印象。

## 先定一句方向描述

如果你现在还说不清，可以先给自己的页面写一句方向描述。比如：

```text
我想要一个简约专业、信息层次清楚、带一点轻微科技感的个人主页。
```

或者：

```text
我希望首页看起来温暖友好、容易交流，不要太像简历，也不要太像模板站。
```

这一步看起来像“定感觉”，其实非常重要。因为一旦方向明确，后面你让 AI 修改排版、颜色、层次时，就不再是在空中挥手，而是在朝着一个已经说得出口的方向收敛。
