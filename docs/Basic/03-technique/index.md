# 第 3 章：做好第一印象 —— 界面、风格与更有效的需求表达

把项目带回本地之后，最适合先动手优化的，往往不是更复杂的功能，而是首页的第一印象。原因很简单：它最直观，也最容易建立反馈。一个页面是不是像模板、有没有层次、有没有自己的气质，几乎一眼就能看出来。对基础版来说，这是很好的练习场，因为你会在这里第一次明显感受到，自己脑子里的“感觉”其实是可以一步步说给 AI 听的。

第 3 章不是设计理论课，也不是让你一口气学会 UI/UX。它要解决的是更实际的问题：你现在已经有一个能跑的主页了，怎么让它不再像默认生成结果，而更像一个认真做过的作品；以及，当你说“我想要它更简洁一点、更有层次一点、更像我一点”的时候，怎么把这些感觉说得足够清楚，让 AI 真正听懂。

这一章之所以重要，不只是因为页面会变好看，而是因为你会开始建立一种以后一直用得上的能力：把模糊感觉说成可执行的修改方向。后面无论你是补内容、调数字分身，还是准备上线，这个能力都会继续用到。

## 这一章的重点到底是什么

很多人一说到“改界面”，脑子里就会自动浮现一堆设计术语，甚至会担心自己是不是没有审美、没有经验、没有参考。其实基础版不需要你一下掌握这么多。你现在最需要建立的，是一种更朴素但更有力量的能力：你能不能看出页面哪里不对劲，能不能说清自己想要的方向，能不能把修改范围收住，让 AI 朝着那个方向稳定推进。

所以这章会尽量克制。我们不会铺开很多风格名词库，也不会把你拖进复杂设计体系。我们只抓最值得先动的部分：首屏排版、颜色和字体、区块层次与留白。只要这三类地方开始对了，页面就会明显从“默认模板”往“作品”靠过去。

换句话说，这一章不是要你做一个满分页面，而是先做一个有主次、有气质、能让别人一眼看懂“你是谁”的首页。

## 做完这一章，你会得到什么

做完第 3 章，你的首页应该已经有更清晰的气质，首屏更容易看懂，手机端也不会显得太乱。更重要的是，你会第一次明显掌握一种能力：不是只会说“帮我改好看一点”，而是能更具体地告诉 AI，现在的问题是什么，我想改成什么感觉，哪些地方能动，哪些地方先别动。这个能力，后面加内容、调数字分身、准备上线时还会继续用到。
