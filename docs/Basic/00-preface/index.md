# 第 0 章：开始前 —— 你会做出什么，怎么学，卡住怎么办

## 先把边界说清

第 0 章的任务很简单：先把期待立住，把边界说清，也把焦虑降下来。你读完之后，手里应该有两样东西，一张清晰的学习地图，以及一套统一的“卡住就问 AI”流程。至于 Prompt / Context / Agentic 的完整方法论、AI 工具大全、MCP / Skills 的系统讲解，这一章都不会展开，因为它们还不是你现在最需要的东西。

很多人打开 AI 教程时，最常见的不是“不会操作”，而是先被一连串问题卡住：我零基础，真的能开始吗；这是不是又一本换了壳的编程教材；我到底会做出什么；如果中途报错了，我是不是就卡住了。

第 0 章先把这些问题说清楚。

看完这一章，你至少会更确定两件事：这套基础篇到底在教什么，以及你下一步该怎么开始，而不是继续观望。

## 先记住一句话

> 这套基础篇不是在训练你“像程序员那样背知识”，而是在训练你“像做东西的人那样把作品做出来”。

你当然会接触到一些技术词，比如提示词、Git、API、部署、环境变量。

但在基础篇里，这些词的出现顺序只服务一个原则：

**什么时候你真的需要它了，我们再讲到够你继续推进的程度。**

不会一上来先把你拖进一堆抽象概念里，也不会把还属于进阶版的系统工程内容提前塞给你。

## 怎么读更顺手

如果你现在最想知道“这是不是我该学的东西”，优先看 `0.1` 和 `0.2`。如果你最怕中途卡住，就先看 `0.3`。如果你想快速看完整条路线，再看 `0.4`。当然，你也完全可以按顺序从 `0.1` 一路读到 `0.4`。

## 读完就可以出发

如果你看完之后已经有这种感觉——“好，我知道这不是语法课了；好，我知道我最终要做出什么了；好，就算卡住我也知道怎么继续了”——那你就可以进入第 1 章，开始真正做出第一个版本。
