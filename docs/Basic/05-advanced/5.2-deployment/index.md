# 5.2 写第一版“数字分身说明书”，并用 1-3 组真实材料校准它

既然数字分身最重要的是“像你”，那你就不能只把它交给一个模糊目标，比如“请像我一样回答问题”。这种说法太空了。真正让它开始变稳的，是一份边界清楚的说明书。你要告诉它你是谁，你主要知道什么，你希望它怎么说话，不知道的时候该怎么回应，以及哪些内容它不应该编。

## 说明书里最该写清什么

这里最容易犯的错误，是把说明书写成一堆抽象性格词，比如“真诚、温暖、专业、有逻辑”。这些词不是完全没用，但它们通常不够落地。真正能让数字分身更像你的，往往不是这些形容词，而是更具体的事实和表达习惯。比如你平时怎么介绍自己，你更喜欢先讲结论还是先讲背景，你会不会用某些固定的句式，你遇到不知道的问题时，是更倾向于坦白说不知道，还是先说明自己知道到哪一步。

一份第一版说明书，至少应该说清下面几件事：

| 要写什么 | 为什么重要 |
|------|------------|
| 你是谁 | 决定它最基本的身份和事实边界 |
| 你知道什么 | 决定哪些问题它可以稳定回答 |
| 你怎么说话 | 决定回答的语气和表达风格 |
| 你不知道时怎么办 | 决定它会不会乱编 |

## 一个很够用的第一版骨架

你不需要把它写成复杂文档，可以先从这种骨架开始：

```text
你是我的数字分身，用来在个人主页里回答访客关于我的问题。

你的任务：
- 介绍我是谁
- 回答和我有关的问题
- 帮访客了解我最近在做什么、做过什么、怎么联系我

关于我：
- 我是：______
- 我最近在做：______
- 我擅长或长期关注：______

说话方式：
- 语气：______
- 回答尽量：简洁 / 真诚 / 人话一点 / 不装专家

边界：
- 不要编造我没做过的经历
- 不要假装知道我没提供的信息
- 不知道时要明确说不知道，并建议访客通过联系方式进一步确认
```

写到这个程度，就已经比一句“请像我一样回答”强很多了。

## 真正让它更像你的，不是形容词，而是样本

所以这一节除了写说明书本身，还要做一件同样关键的事：用 1 到 3 组真实材料去校准它。这里完全不需要很多。几段你平时真的会说的话，一两组你介绍自己的真实问答，往往比一长串抽象要求有用得多。因为模型最容易模仿的，不是“请更有人味一点”这种空指令，而是已经发生过的具体表达。

比如你可以给它这种材料：

```text
示例 1
问：你现在主要在做什么？
答：我最近主要在整理自己的作品，也在尝试用 AI 做一些更完整的小项目。

示例 2
问：你擅长什么？
答：我比较擅长把复杂问题讲清楚，也比较关注 AI 应用、内容表达和知识整理这几个方向。
```

1 到 3 组这样的样本，就已经很有用了。重点不在多，而在真实。

## 这一步其实也是在整理边界

你也可以把这一节理解成一次边界整理。说明书越清楚，后面答偏、编造、风格失真的概率就越低。你不需要把它写成华丽文件，只要它足够清楚，能告诉数字分身：哪些是事实，哪些是语气，哪些问题可以答，哪些问题该收住。到这个程度，它就已经开始从“会回话”慢慢变成“会代表你”。
