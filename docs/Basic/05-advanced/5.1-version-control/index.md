# 5.1 为什么“像你”比“更聪明”更重要，以及先用一句人话理解背后发生了什么

很多人一提到 AI，就会下意识追问：它够不够聪明，模型强不强，能不能回答更复杂的问题。但对个人主页里的数字分身来说，真正重要的问题其实不是“它有多强”，而是“它是不是在代表你”。如果一个回答看起来很完整，却完全不像你的表达方式，也不尊重你的边界，那它再流畅，也很难说是一个好的数字分身。

所以这一章一开始，要先把评价标准拉回来。你的数字分身不是通用百科，也不是全知全能客服。它最重要的任务，是介绍你、回答和你有关的问题，并在不知道的时候诚实收住。只要这个标准清楚了，你后面做的很多调整就会变得更简单：你不是在追求“更万能”，而是在追求“更像你、更稳、更可信”。

## 为什么“像你”更重要

想象一下两种回答。

第一种很标准，很完整，但像任何网站上的默认机器人都会说的话。第二种也许没有那么炫，但会用接近你的方式介绍你，知道哪些能说，哪些不能乱说，不知道的时候会老实收住。对个人主页来说，后者更有价值。

因为别人来到这里，不是为了测试模型智商，而是为了认识你。

## 先用一句人话理解背后发生了什么

这里也很适合先用一句人话理解背后到底发生了什么。最简单的描述就是：访客在页面里输入问题，你的项目把这个问题发给模型接口，模型生成回答，再把回答返回到页面显示出来。你现在不需要立刻理解完整后端架构，也不需要把 HTTP、路由、请求细节全部吃透。你只要先建立一个够用直觉：页面里这个聊天过程，并不是魔法，而是一条可以被你逐步控制的链路。

你可以先把它想成这样：

```text
访客提问
-> 项目把问题送出去
-> 模型结合说明书生成回答
-> 回答回到页面显示出来
```

第 5 章真正要调的，很多时候都发生在这条链路的中间：你给了什么信息，你立了什么边界，你怎样要求它说话。

## “数字分身说明书”是什么

在这条链路里，最关键的一个东西，就是后面我们会写到的“数字分身说明书”。你可以先把它理解成系统提示词，或者更口语一点：一份告诉它“你是谁、你该怎么说、你知道什么、不知道什么时该怎么做”的说明书。

有了这层理解，第 5 章后面所有动作都会落得更实。你就不会把它看成一个神秘的黑箱，而会知道：原来我是在不断改这份说明书、补这份说明书、校准这份说明书。
